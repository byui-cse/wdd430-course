# WDD 430 Course Summary

## Course Overview

WDD 430 is a full-stack web development course focused on modern web application design and delivery. Students practice both frontend and backend development, including API design, data modeling, authentication, and deployment workflows.

The course is intentionally project-centered and team-based. Rather than only consuming lectures, students build, compare, and iterate on real application ideas over the semester.

## Instructional Approach

The course uses a mentor-style model where the instructor coaches teams through applied work. Learning is organized around the BYU-Idaho learning model architecture used in this course:

- **Prepare:** Students come to team meetings with research and progress from assigned work.
- **Teach One Another:** Teams are expected to learn collaboratively and rely on each member's strengths.
- **Ponder/Prove:** Students reflect on what they are learning and validate understanding through experimentation and implementation.

A central expectation is that students move beyond passive reading by building, testing, comparing, and documenting their decisions.

## Course Arc and Sequence

The semester follows a two-phase structure:

1. **Specification and Bakeoff Phase (first ~8 weeks):**
   - Students create project specifications.
   - Teams run multiple "bakeoffs" using different stacks/architectures.
   - The class evaluates tradeoffs and selects a stack for final implementation.
2. **Implementation Phase (last ~5 weeks):**
   - Teams of 3-4 build and deliver a final full-stack project.
   - Emphasis is placed on project management, team communication, and iterative improvement.

Unit-level progression reinforces this arc:

- **Unit 1:** Team formation, setup, architecture/performance research, and project specification.
- **Unit 2:** Comparative stack bakeoffs and evidence-based technology selection.
- **Unit 3:** Final team project development, testing, technical documentation, and post-mortem reflection.

## Learning Outcomes

By the end of the course, students are expected to:

1. Articulate key parts of the full-stack web ecosystem.
2. Develop full-stack web applications using modern tooling and workflows.
3. Use contemporary development and deployment technologies effectively.
4. Deploy applications to cloud infrastructure.
5. Demonstrate productive team practices: communication, collaboration, accountability, and delivery.

## Pedagogical Principles

The course emphasizes:

- **Authentic, applied learning:** Build real artifacts instead of isolated exercises.
- **Comparative evaluation:** Use bakeoffs to examine strengths/weaknesses of technologies in context.
- **Reflection + experimentation:** Validate ideas by implementing and measuring, then reflect on outcomes.
- **Team professionalism:** Use Git/GitHub, planning tools, and clear communication norms.
- **Design-before-build thinking:** Produce specifications that define user goals, MVP scope, models, and API needs.

## Assessment Philosophy

Assessment is portfolio-oriented and outcome-aligned. Students compile evidence across the semester and self-assess against course outcomes, supported by completed artifacts, team work, and project deliverables.

The grading model values initiative, depth of understanding, and practical integration of concepts, not checklist completion.

## AI Use in This Course

AI is framed as a learning support tool, not a replacement for student thinking or work.

- General tutoring/research use is allowed.
- Assignment-specific limits are communicated with stoplight indicators (green/yellow/red).
- Students are expected to review, verify, and adapt AI-generated output.

## Tools and Workflow Expectations

Students are expected to operate in a modern web workflow with:

- VS Code (recommended)
- Node.js (LTS)
- Git and GitHub
- Team communication and coordination tools (for example, Microsoft Teams and Trello)

Course activities also stress understanding package management, build tooling, and task automation as part of professional practice.

## Summary

WDD 430 is a studio-style, team-centered full-stack course where students learn by planning, prototyping, comparing, and shipping. Its pedagogy combines preparation, collaboration, reflection, and proof-through-building to develop both technical depth and professional software practices.
